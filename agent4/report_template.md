## 3. 检测背景

PX4 是广泛应用于无人机、无人车和机器人系统中的开源飞控软件。由于其直接参与飞行控制、传感器融合、任务规划、通信链路和执行机构控制，一旦软件中存在安全缺陷，可能导致飞行异常、任务中断、设备失控，甚至造成物理安全风险。

因此，对 PX4 进行软件安全威胁检测具有重要意义，尤其是在以下场景中：

* 无人机飞控系统部署前的安全评估；
* 开源供应链安全检测；
* 嵌入式软件漏洞发现；
* 飞控关键模块的代码质量审计；
* 自动化安全检测 Agent 的验证实验。

## 6. 检测方法

本次检测采用静态分析、规则匹配、人工复核相结合的方式。

### 6.1 静态代码分析

使用静态分析工具对 PX4 源码进行扫描，重点关注以下问题：

* 数组越界；

### 6.2 安全规则检测

基于自定义安全规则，对源码进行模式匹配，重点识别：

* 危险函数调用；

* 文件路径拼接不安全；
* 配置文件解析风险；
* 命令执行风险；
* 不安全的默认配置；
* 调试接口暴露；
* 构建脚本中潜在的供应链风险。

### 6.3 供应链安全分析

针对 PX4 项目的依赖组件、构建脚本和外部工具链进行初步检查，重点分析：

* 第三方依赖来源是否可信；
* 是否固定依赖版本；
* 是否存在远程下载脚本；
* 构建过程中是否执行外部命令；
* 是否存在未校验完整性的外部资源；
* 是否存在潜在的依赖混淆风险。

### 6.4 人工复核

## 9. 详细问题列表

| ---- | ---------------------------------- |
| 风险等级 | 高危 / 中危 / 低危                       |
| 问题类型 | 例如：数组越界 / 空指针引用 / 输入校验不足           |

#### 问题描述

在 `xxx` 模块中，程序对外部输入或内部变量缺少充分校验，可能在特定条件下触发异常行为。例如，当输入数据为空、数组索引超出范围或函数返回失败时，代码仍继续执行，可能导致飞控任务异常、模块崩溃或控制逻辑错误。

#### 风险影响

该问题可能造成以下影响：

* 导致模块异常退出；
* 影响飞控系统稳定性；
* 造成错误的控制决策；
* 被异常输入触发后可能导致拒绝服务；
* 在关键任务阶段可能扩大为飞行安全风险。

#### 相关代码

```cpp
// 示例代码
if (data[index] > threshold) {
    execute_control();
}
```

#### 触发条件

1. 攻击者或异常环境能够影响输入数据；
2. 输入数据未经充分校验；
3. 程序进入相关控制分支；
4. 异常条件被触发后缺少安全处理逻辑。

#### 修复建议

建议在访问数组、指针、外部输入和函数返回值前增加校验逻辑。例如：

```cpp
if (index >= 0 && index < data_size) {
    if (data[index] > threshold) {
        execute_control();
    }
} else {
    PX4_ERR("Invalid index");
    return;
}
```

同时建议：

* 增加边界检查；
* 检查函数返回值；
* 对异常输入进行过滤；
* 增加日志记录；
* 在关键控制逻辑中设置安全降级策略。

## 10. 典型问题分析

### 10.1 输入校验不足

PX4 中大量模块依赖传感器数据、通信消息和参数配置。如果输入数据缺少边界检查，可能导致异常数据进入控制逻辑，影响飞控系统的稳定性。

建议对以下输入加强校验：

* MAVLink 消息字段；
* 传感器原始数据；
* 参数配置值；
* 文件读取结果；
* 数组索引；
* 指针对象；
* 外部命令返回结果。

### 10.2 错误处理不足

部分代码可能存在函数调用后未检查返回值的问题。如果底层接口调用失败，而上层逻辑继续执行，可能导致后续状态异常。

建议统一采用以下处理方式：

```cpp
int ret = function_call();

if (ret != PX4_OK) {
    PX4_ERR("function_call failed: %d", ret);
    return ret;
}
```

### 10.3 构建流程供应链风险

PX4 项目的构建过程依赖多个脚本、工具链和外部组件。如果构建脚本中存在未固定版本、远程下载或未校验完整性的资源，可能带来供应链污染风险。

建议：

* 固定依赖版本；
* 使用可信源下载依赖；
* 对下载文件进行哈希校验；
* 避免在构建脚本中执行不可信命令；
* 对 CI/CD 流程增加安全扫描。

## 13. 安全加固建议

为了进一步提升 PX4 软件安全性，建议从以下几个方面进行加固：

1. **建立安全编码规范**

6. **增加安全测试用例**
